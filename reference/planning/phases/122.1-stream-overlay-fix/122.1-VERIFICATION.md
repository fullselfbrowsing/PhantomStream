**Note on timing:** broadcastOverlayState reads `FSB.actionGlowOverlay.targetElement` at call time. In show(), the broadcast fires AFTER `_startTracking()` and the targetElement is set, so the rect is valid. In hide()/destroy(), targetElement is explicitly set to null BEFORE the broadcast, which means broadcastOverlayState finds no glowSource and sends `glow: null`. The dashboard then hides the glow rect. This is the correct intended behavior.
