**Root cause analysis (read before coding):**

The UAT found three interconnected bugs:
- handleStopAutomation has no re-entry guard. While cleanupSession awaits loopPromise (up to 5s), re-entrant calls find the session still in activeSessions and re-process it.
- handleStopAutomation does not send an `automationComplete` chrome.runtime message, so the `completionListener` inside executeAutomationTask never fires. The promise only resolves on the safety timeout (up to 4 minutes later).
- Because startDashboardTask's `await executeAutomationTask(...)` hangs, `broadcastDashboardComplete` never runs on the stop path. The only ext:task-complete comes from _handleStopTask's callback, but the repeated stops and 5s cleanup await make delivery unreliable.

**Fix A: Idempotency guard in handleStopAutomation (line 6705)**

At the top of `handleStopAutomation`, after the sessionId fallback block (after line 6711), add:

```javascript
// Idempotency guard: skip if this session is already being stopped
if (session && (session.status === 'stopped' || session._stopInFlight)) {
  automationLogger.info('Stop already in progress, skipping duplicate', { sessionId });
  sendResponse({ success: true, message: 'Stop already in progress', duplicate: true });
  return;
}
```

Then immediately after the `if (session)` block at line 6740, set the flag:

```javascript
if (session) {
  session._stopInFlight = true;  // <-- ADD THIS LINE (before session.status = 'stopped')
  automationLogger.debug('Found session to stop', { sessionId, status: session.status });
  session.status = 'stopped';
  // ... rest unchanged ...
```

This must go BEFORE `session.status = 'stopped'` because there's a window between finding the session and setting the status where a second call could slip through. The `_stopInFlight` flag closes that window.

IMPORTANT: The idempotency check must happen AFTER the session is looked up (after the storage fallback at line 6738), not before. The check needs the `session` variable.

**Fix B: Send automationComplete on stop so executeAutomationTask promise resolves**

Inside handleStopAutomation, AFTER `session.status = 'stopped'` and BEFORE `await cleanupSession(sessionId)` (i.e., between the lastAction capture and the cleanupSession call), add:

// Resolve the executeAutomationTask promise so startDashboardTask can complete.
// Without this, the completionListener inside executeAutomationTask never fires
// and the promise hangs until the safety timeout (up to 4 minutes).

This sends the message that the `completionListener` (line 6544) is waiting for. The listener resolves the `executeAutomationTask` promise, which means `startDashboardTask` can proceed to call `broadcastDashboardComplete`. The `_dashStopSent` dedup flag (already in place) will then correctly skip the duplicate.

Replace the `_handleStopTask()` method body with:

```javascript
_handleStopTask() {
  // Idempotency: ignore if a stop is already in-flight
  if (this._stopInFlight) {
    console.log('[FSB WS] Stop already in-flight, ignoring duplicate dash:stop-task');
    return;
  }
  this._stopInFlight = true;

  console.log('[FSB WS] Stop task received from dashboard');

  // Per D-03: stop ANY running automation, not just dashboard tasks
  // Mirror the mcp:stop-automation pattern from background.js
  this._dashStopSent = true;
  handleStopAutomation(
    { action: 'stopAutomation' },
    { id: chrome.runtime.id },
    (result) => {
      console.log('[FSB WS] Stop result:', JSON.stringify(result));

      // Skip sending if this was a duplicate (handleStopAutomation already handled it)
      if (result && result.duplicate) {
        console.log('[FSB WS] Duplicate stop -- not sending ext:task-complete again');
        this._stopInFlight = false;
        return;
      }

      if (result && result.success) {
        this.send('ext:task-complete', {
          success: false,
          error: 'Stopped by user',
          elapsed: result.duration || 0,
          stopped: true,
          lastAction: result.lastAction || null
        });
      } else {
        // No session found or already stopped -- still acknowledge
        this.send('ext:task-complete', {
          success: false,
          error: 'Stopped by user',
          elapsed: 0,
          stopped: true,
          lastAction: null
        });
      }

      // Reset for next task
      this._stopInFlight = false;
    }
  );
}
```

Key changes from the plan-01 implementation:
- Added `_stopInFlight` guard at the top -- returns immediately if a stop is already being processed
- In the callback, checks `result.duplicate` (from Task 1's idempotency guard in handleStopAutomation) and skips sending ext:task-complete if it's a duplicate response
- Resets `_stopInFlight = false` in both the duplicate and normal callback paths so the next task can be stopped
