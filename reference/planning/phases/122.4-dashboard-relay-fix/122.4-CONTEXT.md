# Phase 122.4: Dashboard Relay Fix - Context

**Gathered:** 2026-03-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix three connected failures: (1) task completion result never reaches dashboard, (2) DOM stream not rendering in preview, (3) stream is computationally expensive and unresponsive. Root causes: missing .catch() on automationComplete sendMessage, potential completionListener removal race, and capturing ALL 300+ computed styles on every element crushing performance.

</domain>

<decisions>
## Implementation Decisions

### Task result delivery (CRITICAL)
- **D-01:** Add .catch(() => {}) to chrome.runtime.sendMessage at line 11715 (automationComplete on normal completion). Without it, the unhandled rejection can kill the code path in MV3 service worker.
- **D-02:** Add console.log before and after the automationComplete sendMessage to trace whether it fires and whether the completionListener catches it.
- **D-03:** Add a fallback: if broadcastDashboardComplete hasn't fired within 5s of "Task completed successfully", send ext:task-complete directly from a setTimeout guard in startDashboardTask.

### Stream performance (CRITICAL)
- **D-04:** Revert from ALL 300+ computed styles back to a curated list. The full iteration is too expensive on heavy pages (YouTube DOM fetch took 45s). Use an expanded curated list of ~80-90 properties (the original 66 + the most impactful layout/visual properties) instead of iterating getComputedStyle.length.
- **D-05:** Throttle domStreamOverlay broadcasts. Currently fires on every rAF tick during automation (40+ per session). Throttle to max 1 per 500ms.
- **D-06:** Add a snapshot size budget. If serialized HTML exceeds 200KB, skip computed styles on non-visible/below-fold elements. Prioritize viewport elements.

### Stream rendering
- **D-07:** The compression (Phase 122.3) + cache fix (Phase 122.4 first commit) should have fixed rendering. Verify after deploying the above fixes. If still broken, add debug logging to dashboard ws.onmessage to trace exactly where messages are lost.

### Claude's Discretion
- Exact property list for the curated ~80-90 styles (expand the 66 with most impactful additions)
- Whether to use a WeakMap cache for computed styles to avoid re-computing on unchanged elements
- Overlay throttle interval (500ms vs 1000ms)

</decisions>
