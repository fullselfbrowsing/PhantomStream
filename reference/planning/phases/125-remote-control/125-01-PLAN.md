Add remote control infrastructure to background.js. Insert after the `_tabSwitchTimer` variable declaration (around line 824):

1. Add a new state variable:

var _remoteControlDebuggerTabId = null; // Tab ID with debugger attached for remote control (null = not active)

2. Add three handler functions BEFORE the `executeCDPToolDirect` function (around line 12440). These are called by ws-client.js message handlers:

**handleRemoteControlStart():**
- Gets `_streamingTabId` as the target tab. If null, log warning and return.
- Attaches `chrome.debugger` to `_streamingTabId` with protocol '1.3'. If already attached (error message includes 'Already attached'), that is OK -- log and continue.
- Sets `_remoteControlDebuggerTabId = _streamingTabId`.
- Logs `[FSB Remote] Debugger attached to tab ${_streamingTabId} for remote control`.

**handleRemoteControlStop():**
- If `_remoteControlDebuggerTabId` is null, return silently.
- Detaches `chrome.debugger` from `_remoteControlDebuggerTabId` in a try/catch (ignore errors -- tab may have closed).
- Sets `_remoteControlDebuggerTabId = null`.
- Logs `[FSB Remote] Debugger detached -- remote control stopped`.

**handleRemoteClick(payload):**
- Expects payload: `{ x: number, y: number, button: 'left'|'right', modifiers: number }`.
- If `_remoteControlDebuggerTabId` is null, log warning `[FSB Remote] Click ignored -- remote control not active` and return.
- Use `chrome.debugger.sendCommand({ tabId: _remoteControlDebuggerTabId }, 'Input.dispatchMouseEvent', ...)` exactly like the cdpClickAt pattern:
  - mousePressed with `{ type: 'mousePressed', x: payload.x, y: payload.y, button: payload.button || 'left', clickCount: 1, modifiers: payload.modifiers || 0 }`
  - mouseReleased with matching params
- Log `[FSB Remote] Click dispatched at (${payload.x}, ${payload.y})`.
- Wrap in try/catch. On error: log `[FSB Remote] Click failed: ${err.message}`, set `_remoteControlDebuggerTabId = null` (debugger may have detached).

**handleRemoteKey(payload):**
- Expects payload: `{ type: 'keyDown'|'keyUp'|'char', key: string, code: string, text: string, modifiers: number }`.
- If `_remoteControlDebuggerTabId` is null, log and return.
- For type 'char': use `chrome.debugger.sendCommand({ tabId: _remoteControlDebuggerTabId }, 'Input.dispatchKeyEvent', { type: 'char', text: payload.text || payload.key })`.
- For type 'keyDown': use `Input.dispatchKeyEvent` with `{ type: 'keyDown', key: payload.key, code: payload.code, text: payload.text || '', modifiers: payload.modifiers || 0 }`.
- For type 'keyUp': use `Input.dispatchKeyEvent` with `{ type: 'keyUp', key: payload.key, code: payload.code, modifiers: payload.modifiers || 0 }`.
- Log `[FSB Remote] Key dispatched: ${payload.type} ${payload.key}`.
- Wrap in try/catch. On error: log, set `_remoteControlDebuggerTabId = null`.

**handleRemoteScroll(payload):**
- Expects payload: `{ x: number, y: number, deltaX: number, deltaY: number }`.
- If `_remoteControlDebuggerTabId` is null, log and return.
- Use `chrome.debugger.sendCommand({ tabId: _remoteControlDebuggerTabId }, 'Input.dispatchMouseEvent', { type: 'mouseWheel', x: payload.x, y: payload.y, deltaX: payload.deltaX || 0, deltaY: payload.deltaY || 0 })`.
- No logging for scroll (too frequent -- would spam console).
- Wrap in try/catch. On error: log, set `_remoteControlDebuggerTabId = null`.

Important: These functions do NOT attach/detach the debugger per call. The debugger is attached once in handleRemoteControlStart and detached in handleRemoteControlStop. This avoids the overhead of attach/detach on every mouse click or keystroke.

Important: If `executeCDPToolDirect` is called during automation while remote control is active, it will get 'Already attached' on its attach call -- this is already handled (lines 12447-12454). No conflict.
