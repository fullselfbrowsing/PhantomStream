| # | Truth | Status | Evidence |

### Gaps Summary

No gaps. All four must-haves are fully implemented, wired, and verified:

- `lib/lz-string.min.js` and `showcase/js/lz-string.min.js` exist, are byte-identical, and pass a round-trip smoke test (2568 -> 304 -> 2568 bytes, 12% of original size).
- `ws/ws-client.js` `send()` applies threshold-gated compression (>1KB) with the `{ _lz: true, d: compressed }` envelope and a size-check guard.
- `background.js` loads `lib/lz-string.min.js` via `importScripts` at position 8836, before `ws-client.js` at position 8874 -- the ordering guarantee that makes `LZString` available at call time.
- `showcase/dashboard.html` loads `js/lz-string.min.js` before `js/dashboard.js`.
- `dashboard.js` `ws.onmessage` detects the `_lz` flag, decompresses, parses, and dispatches correctly; the else path preserves backward compatibility for uncompressed messages.
- Pings bypass the `send()` method entirely (direct `this.ws.send()` call in `_startKeepalive`), satisfying the "no overhead for small control messages" must-have.
- Both commits (`fd4f748`, `53ac78b`) exist in git history.

Three items require human verification with a live browser session but these are runtime behaviors that cannot be verified statically. The code path from compression to decompression to iframe rendering is fully wired in the codebase.
