## Decisions Made
- 1KB threshold: pings (~50 bytes), task-progress (~200 bytes) skip compression; dom-snapshots (100KB+) get compressed. Avoids overhead for small control messages.
- Envelope format { _lz: true, d: compressed }: the relay server (fly.io passthrough) sees a small JSON object instead of the huge payload that was being dropped.
- Vendored library in two locations (lib/ for extension, showcase/js/ for dashboard) to keep each context self-contained and avoid cross-directory path issues.
- Size guard: if compression somehow makes the payload larger, send uncompressed. Unlikely for 1KB+ JSON but handles edge cases.
