<objective>
Make the optional-hls.js contract real in the package manifest and PROVE the zero-hard-dependency guarantee. Add hls.js to `package.json` under `peerDependencies` + `peerDependenciesMeta.optional:true` (NEVER `dependencies`), verify `dependencies` stays exactly `{ ws }`, and add an explicit assertion to `scripts/package-smoke.mjs` (and a unit check in `tests/package-publish.test.js`) that `./renderer` imports cleanly with hls.js absent — which holds only because Plan 02 made the hls.js import a dynamic `import('hls.js')` inside a function, never a top-level import.

Purpose: This closes MADPT-01's zero-runtime-dep half. The dynamic-import-only rule is the single most important packaging constraint of the phase: `package-smoke.mjs` already `await import('./renderer')`s every export subpath in a sandbox where hls.js is NOT installed; a top-level `import 'hls.js'` would fail that import with ERR_MODULE_NOT_FOUND. An OPTIONAL peerDependency means npm will not auto-install hls.js and will not warn when it is absent — exactly the host-controlled, zero-hard-dep posture.

Output: the package.json packaging edit + the smoke assertion + a publish-test assertion.
</objective>
