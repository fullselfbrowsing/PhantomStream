<objective>

The dynamic-import-only rule for hls.js is load-bearing: a top-level `import 'hls.js'` would break `npm run package:smoke` (Plan 05) and the zero-hard-dep guarantee.

Output: `src/renderer/media-player.js` (new), the `media-unavailable` overlay in `overlays.js`, the `blob:` CSP add in `snapshot.js`, and the player + CSP tests.

<decision_tree>
<!-- 14-RESEARCH "The attachment decision tree" — implement EXACTLY this order in attach(). -->
attach(videoEl, manifestUrl, ctx):  // videoEl lives in iframe.contentDocument
  1. GATE: if !gateAsset(manifestUrl,'media').allow -> degrade(nid,'no-manifest'); return
  2. NATIVE HLS FIRST: if classifyManifest({url:manifestUrl}) === 'hls'
        AND videoEl.canPlayType('application/vnd.apple.mpegurl') in {'probably','maybe'}:
        videoEl.src = manifestUrl; (Plan 03 calls ensurePlaying); return handle{kind:'native'}
  3. HOST playerFactory (if provided): const player = playerFactory(ctx);
        player.attach(videoEl, manifestUrl, ctx); wire player.onError -> degrade(reasonFromError);
        return handle{kind:'factory', player}
  4. OPTIONAL LAZY hls.js (HLS only, no factory):
        if !(win.ManagedMediaSource || win.MediaSource) -> degrade('mse-opaque'); return
        const Hls = await tryLazyImportHls();
        if !Hls || !Hls.isSupported() -> degrade('no-player'); return
        const hls = new Hls({ /* emeEnabled stays FALSE */ });
        videoEl.addEventListener('encrypted', () => degrade('drm'), {once:true});
        hls.on(Hls.Events.ERROR, (_e,d) => { if (d.fatal) degrade(d.type===Hls.ErrorTypes.KEY_SYSTEM_ERROR ? 'drm' : 'mse-opaque'); });
        hls.loadSource(manifestUrl); hls.attachMedia(videoEl);  // LOAD then ATTACH
        return handle{kind:'hls', hls}
  5. DASH (.mpd) with no factory -> degrade('no-player')
  6. anything unhandled -> degrade('no-manifest')
  WRAP the whole body in try/catch -> degrade('mse-opaque') on ANY throw. NEVER rethrow.

tryLazyImportHls(): try { const m = await import('hls.js'); return m && (m.default||m.Hls||m) || null; } catch { return null; }
  // DYNAMIC import inside the function ONLY — never a top-level import (breaks package:smoke / zero-hard-dep).

degrade(nid, reason): destroy any live player for nid (revoke object URLs in the PARENT, removeAttribute('src')+load() on the child); show 'media-unavailable' overlay over resolveNidRect(nid); invoke onMediaUnavailable(nid, reason) via the contained hook; keep the element's poster if present else the Phase-12 placeholder. Reason codes: no-manifest | no-player | mse-opaque | drm.
</decision_tree>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent realm ↔ child iframe (`allow-same-origin`, no `allow-scripts`) | The parent-realm player binds a `blob:` object URL / native manifest to the inert in-iframe `<video>`; the child runs no script |
| viewer → manifest/segment origin | The player fetches manifest/segments in the PARENT realm from attacker-influenced URLs (re-gated in Plan 03; here the player accepts an already-gated url + a `gateAsset` dep) |
| attacker-influenced media element | A `<video>` may carry encrypted/DRM content or an opaque MSE source — must never break the mirror |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04 | Elevation (XSS) | Adaptive player running inside the mirror sandbox | mitigate | Player runs ONLY in the parent realm (`createMediaPlayer` is renderer-owned); the iframe sandbox stays exactly `allow-same-origin`, NO `allow-scripts` (asserted in renderer tests, re-asserted Plan 03/Phase 15). The child receives only a `blob:` string / manifest src. |
| T-14-05 | Info-disclosure | DRM/EME protected content (license/keys) mirrored | mitigate | NEVER attempt EME: `emeEnabled` stays false; the child `'encrypted'` event and hls.js `KEY_SYSTEM_ERROR` both `degrade('drm')` — Task 1 tests prove both paths. |
| T-14-06 | Tampering / Elevation | Parent-realm object-URL blast radius (leaked/orphaned blob URLs) | mitigate (surface documented; full model Phase 15/MSEC-04) | Mint AND revoke object URLs in the parent; `degrade`/`destroy` revoke + `removeAttribute('src')` + `load()`; `destroyAll()` on re-snapshot (Plan 03). 14-RESEARCH records the surface; Phase 15 completes the threat model. |
