<!-- 14-RESEARCH "Renderer side" + "The manifest→element correlation problem". -->
handleMediaHint(payload):
  1. isCurrentStream(payload, activeIdentity) — else drop (staleness guard).
  2. re-gate: if !gateAssetUrl(payload.manifestUrl, assetCtx).<allow> -> degrade(nid|page-target, 'no-manifest'); return. (V12/SSRF defense in depth — the hint is attacker-influenced.)
  3. scope 'element' (payload.nid set) AND the element resolves AND is MSE-opaque/source-less:
       mediaPlayer.attach(el, payload.manifestUrl, ctx{ nid, kind: payload.kind, doc, videoEl: el, gateAsset, logger });
  4. scope 'page' (nid omitted): store in a small per-viewer pendingHints structure (most-recent-wins per kind); when an MSE-opaque element next starts (handleMedia/play path with no resolvable source), consume the most-recent matching page hint and attach.
  // Never block on perfect correlation (locked). A page hint with no consumer just sits until an opaque element plays or the next snapshot clears it.
</hint_consumption_spec>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wire (STREAM.MEDIA_HINT) → viewer | An attacker-influenced `manifestUrl` arrives in a hint payload and must be re-gated before any use |
| parent realm ↔ child iframe | The player binds blob:/manifest to the inert child `<video>`; players must be torn down on re-snapshot to avoid dead-element/object-URL leaks |
| host config callbacks | `onMediaUnavailable`/`playerFactory` are host-supplied and may throw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-10 | Tampering / Info-disclosure (V12/SSRF) | handleMediaHint fetching an attacker manifest/segment URL | mitigate | `handleMediaHint` re-gates `manifestUrl` with `gateAssetUrl` (https-only + private-range deny, fail-closed) BEFORE binding; a blocked origin → `degrade('no-manifest')`, never fetched (Task 1 blocked-origin test). |
| T-14-11 | Tampering | A stale/late hint from a prior page binds the wrong media | mitigate | `isCurrentStream(payload, identity)` staleness guard drops mismatched streamSessionId/snapshotId (Task 1 staleness test); the hint is identity-stamped (Plan 01). |
| T-14-06 | Elevation / Tampering | Orphaned parent-realm object URLs / dead-element players across re-snapshots | mitigate (surface documented; full model Phase 15) | `mediaPlayer.destroyAll()` on every new-identity snapshot revokes parent object URLs + frees players before the document swap (Task 2; Pattern 2 / Pitfall 2). |
