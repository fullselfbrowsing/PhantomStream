| Token | Value | Usage |
|-------|-------|-------|

Exceptions:
- `1px` borders for panes, buttons, URL blocks, and status strips.
- `2px` focus ring and glow border, inherited from Phase 2 overlay parity.
- `6px` button, badge, URL block, and row radius.
- `8px` status strip radius.
- `12px` pane/stage radius, inherited from the approved loopback demo.
- Interactive controls must have `min-height: 36px`; compact status badges are non-interactive and may be shorter.

## Two-Tab Demo Surface

### Terminal Output

`npx phantom-stream demo` prints URLs as the deterministic default. The CLI output is part of the user experience contract because it is the first step of the demo.

Required lines:

```text
PhantomStream demo running on 127.0.0.1
Source tab: {sourceUrl}
Viewer tab: {viewerUrl}
Room: {roomKeyPrefix}â€¦
```

Rules:
- Print the actual bound ports after next-free-port fallback.
- Use `127.0.0.1` in URLs, not a LAN host.
- Do not auto-open the browser unless a CLI flag asks for it.
- Do not print a marketing paragraph or setup walkthrough.

### Shared Page Shell

Both source and viewer tabs use the Phase 2 demo shell:
- Body: `background: #0f1117; color: #e0e0e0; padding: 24px; max-width: 1200px; margin: 0 auto`.
- Header: title left, compact status badge right, wraps cleanly under narrow widths.
- Subtitle: 15px body text, `#888888`, max one short sentence.
- Main content: one primary pane per tab, not a split landing page.
- Footer/log: mono, 13px, `#888888`, only for transport/control proof lines.

### Source Tab

Purpose: a small live page that produces visible DOM mutations.

Required layout:
- Title: `PhantomStream â€” Source Tab`
- Subtitle: `Mutate this page; the viewer tab mirrors it through the local relay.`
- Header badge: `CAPTURING` while capture is active, `PAUSED` only when auto-mutate is off; this badge is demo state, not relay lifecycle.
- Top status strip: room prefix, relay connection state, sent frame count, last send time.
- Pane header: `SOURCE`
- Controls, in order: `Add row` primary, `Remove row`, `Edit text`, `Show dialog`, `Auto-mutate`.
- Row list starts with at least three rows so the first snapshot has visible content.

Interaction:
- `Add row` appends a row and emits a visible mutation.
- `Remove row` removes the last row only; no confirmation.
- `Edit text` changes one existing row.
- `Show dialog` calls native `alert()` to prove dialog mirroring.
- `Auto-mutate` is a toggle with `aria-pressed`; default on; interval is about 1000ms.

### Viewer Tab

Purpose: the live mirror and compact connection/health proof.

Required layout:
- Title: `PhantomStream â€” Viewer Tab`
- Subtitle: `Live mirror from the source tab through the local relay.`
- Header badge reflects viewer lifecycle exactly: `Connecting`, `Live`, `Stale`, `Disconnected`.
- Top status strip contains lifecycle, room prefix, last frame time, and relay transport state.
- Health row contains only counters/timestamps, never mirrored page content: `frames`, `snapshots`, `mutations`, `misses`, `apply failures`, `drops`, `errors`.
- Mirror stage is the primary visual anchor: `position: relative; aspect-ratio: 16 / 10; min-height: 280px; height: min(72vh, 720px); background: #0f1117; border: 1px solid #333; border-radius: 12px; overflow: hidden`.
- On small screens, keep the stage width at 100% and reduce only height floor to `220px`; do not create horizontal scrolling.

Empty and retained-frame behavior:
- Before the first snapshot, show the empty state centered behind the viewer root.
- After a `stale` or `disconnected` transition, keep the last rendered frame visible if one exists; the compact status strip carries the warning.
- Do not add a full-screen blocking disconnected overlay.

## Lifecycle & Health Interaction Contract

Lifecycle names are exact: `connecting`, `live`, `stale`, `disconnected`.

| State | Visual | Trigger Intent | Demo Copy |
|-------|--------|----------------|-----------|
| `connecting` | Attention dot `#eab308`; no pulse | Initial state before the first accepted frame/snapshot, or during short reconnect window | `Connecting` |
| `live` | Live dot `#22c55e`; pulse allowed when reduced motion permits | First accepted snapshot/frame has arrived and transport is healthy | `Live` |
| `stale` | Attention dot `#eab308`; no pulse | Last frame retained but freshness is suspect: miss threshold, resync pending, relay close/reconnect | `Stale` |
| `disconnected` | Destructive dot `#ef4444`; no pulse | Transport closed beyond the short stale/reconnect window | `Disconnected` |

Timing and motion:
- LIVE dot pulse: 2000ms ease-in-out, opacity `1 -> .6 -> 1`, wrapped in `@media (prefers-reduced-motion: no-preference)`.
- Button hover: 150ms ease.
- State text changes immediately when events arrive; do not debounce state labels.
- Health metric values update at most once per animation frame or on event delivery; avoid timers that visually churn while no frames arrive.

Accessibility:
- The lifecycle badge uses `role="status" aria-live="polite"`.
- Status dots are `aria-hidden="true"`.
- URL blocks and room prefixes are selectable text.
- Focus ring: `2px solid #f59e0b`, offset `2px`, visible on all buttons and links.
