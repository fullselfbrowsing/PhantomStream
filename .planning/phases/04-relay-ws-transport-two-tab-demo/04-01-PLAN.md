<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser tab -> WebSocket backend | Untrusted local browser code connects to `/ws` with room and role query parameters |
| WebSocket backend -> relay core | Raw frame strings cross from socket IO into room routing and diagnostics |
| relay core -> opposite-side clients | Relay fans out raw frame strings to viewers or sources |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing / Access Control | `src/relay/backends/ws.js`, `src/relay/relay.js` | mitigate | Require non-empty room and role `source|viewer`; route only opposite-side frames; invalid role/path sockets are rejected before room attach. |
| T-04-02 | Denial of Service | `src/relay/limits.js`, `src/relay/relay.js` | mitigate | Enforce `RELAY_PER_MESSAGE_LIMIT_BYTES` with `Buffer.byteLength(raw, 'utf8')` before fan-out and record structured `message-too-large` diagnostics. |
| T-04-03 | Denial of Service | `src/relay/relay.js` | mitigate | Drop frames for a target whose `bufferedAmount` exceeds `BACKPRESSURE_BUFFER_LIMIT_BYTES`, count/log the drop, continue delivery to healthy targets. |
| T-04-09 | Spoofing | `src/relay/backends/ws.js` | mitigate | Require generated room key query parameter and reject empty/missing keys; full auth is not introduced in this phase per D-06. |
</threat_model>
