<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local HTTP client -> demo server | Browser requests local fixture/viewer assets and relay upgrade path. |
| demo server -> Playwright driver | Optional `--drive` launches a browser process and injects the adapter. |
| viewer/source endpoints -> relay | Demo endpoints send raw frames through Phase 04 relay. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | control payloads | mitigate | Demo adapter path consumes 05-01 validators through 05-02 adapter. |
| T-05-02 | Information Disclosure | CLI/server logs | mitigate | CLI prints only local URLs, room prefix, and default-deny control state; no mirrored payloads or typed text. |
| T-05-03 | Elevation of Privilege | demo authorization hook | mitigate | Hook returns true only for explicit demo `authorizationMode === 'approve'`; default output remains default-deny. |
| T-05-04 | Elevation of Privilege | denied actions | mitigate | 05-02 adapter active-state check remains in force for every demo action. |
| T-05-05 | Tampering | driver loop/replay | mitigate | Server uses Playwright APIs and adapter replay only; no relay or DOM synthetic event execution. |
| T-05-06 | Information Disclosure | navigation injection | mitigate | Adapter installed before fixture navigation and restarted on navigation by 05-02. |
| T-05-07 | Tampering | coordinate mapping | mitigate | 05-03/05-05 map and reject coordinates before action frames leave viewer UI. |
| T-05-08 | Spoofing | `examples/playwright-demo/server.js` | mitigate | Bind only `127.0.0.1`, reject other hosts, generate room keys, print actual local URLs, and serve no-store assets. |
| T-05-09 | Tampering | relay | mitigate | Reuse `createRelay` raw fan-out; do not add control authorization or action execution to relay files. |
</threat_model>
