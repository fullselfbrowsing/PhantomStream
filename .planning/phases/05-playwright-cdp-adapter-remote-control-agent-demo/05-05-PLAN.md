<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| demo viewer UI -> adapter | Host-owned UI creates control frames that can request native input. |
| mirror stage overlay -> viewport mapping | Browser event coordinates are transformed into real page viewport coordinates. |
| fixture page -> mirror | Captured fixture content is mirrored through the normal capture/relay/viewer path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | viewer control frame creation | mitigate | Use 05-03 mapping and 05-01 action shapes; send nothing for outside/invalid points. |
| T-05-02 | Information Disclosure | action log/health UI | mitigate | Display character counts, coordinates, deltas, state names, and counters only; never raw typed text or mirrored content. |
| T-05-03 | Elevation of Privilege | authorization UI | mitigate | Default segmented control is Deny; `Request control` always sends a request and never marks active locally without adapter state. |
| T-05-04 | Elevation of Privilege | inactive overlay | mitigate | Overlay has pointer events only in `active`; locked/denied/stopped states emit blocked UI and send no action frames. |
| T-05-05 | Tampering | real-page input | mitigate | Viewer sends protocol frames only; 05-02 adapter performs driver-native replay. |
| T-05-06 | Information Disclosure | navigation/resnapshot | mitigate | Fixture is local and non-secret; adapter from 05-02 handles fresh snapshots after navigation. |
| T-05-07 | Tampering | coordinate mapping | mitigate | Use `mapHostPointToViewport` and reject `inside:false` before sending click/scroll. |
| T-05-08 | Spoofing | local demo UI | mitigate | UI consumes local room URL, displays room prefix only, and relies on 05-04 local-only server. |
| T-05-09 | Tampering | relay | mitigate | Viewer uses existing transport; no relay-side control execution is introduced. |
</threat_model>
