# Phase 05 — UI Design Contract

> Visual and interaction contract for the Playwright/CDP adapter remote-control demo only. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Binding context:** preserve Phase 05 decisions: the demo is local, deterministic, operational, and compact; remote control is default-deny; approval/denial is host-hook driven and observable; click/type/scroll feedback is visible without exposing mirrored page content; framework APIs expose state/events but do not impose product UI.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Dot/label gaps, dense metric gaps, action feedback ring offset |
| sm | 8px | Control gaps, segmented-control gaps, badge spacing, log spacing |
| md | 16px | Panel padding, status strip padding, grid gaps, stage empty-state padding |
| lg | 24px | Page padding, header margin, major block gaps |
| xl | 32px | Wide layout section breaks |
| 2xl | 48px | Reserved for large desktop separation only if a secondary diagnostic band is added |
| 3xl | 64px | Unused this phase |

Exceptions:
- `1px` borders for panes, status strips, buttons, segmented controls, and mirror stage.
- `2px` focus ring, active-control stage border, and click feedback ring.
- `4px` radius for transient action feedback rings.
- `6px` button, segmented-control option, row, and feedback chip radius.
- `8px` status strip radius.
- `12px` pane/stage radius, inherited from Phase 04.
- Interactive controls use `min-height: 36px`; no icon-only controls are allowed in this phase.
- On viewports under `640px`, body padding becomes `16px`; all other spacing tokens remain unchanged.

## Typography

Exactly 4 sizes and 2 weights:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 13px | 600 | 1.2 |
| Body | 15px | 400 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Details:
- Page title uses Display.
- Section headings, badges, segmented-control labels, metric labels, and feedback chips use Label, uppercase only for badge/metric labels, `letter-spacing: 1px`.
- Room prefixes, counters, timestamps, coordinates, and log entries use Body size with the mono stack.
- Button text uses Label size, weight 600, line-height 1.2.
- Do not scale fonts with viewport width. Text wraps normally; do not use negative letter spacing.
- Existing renderer overlay internals may retain inherited Phase 02/04 built-in sizing; the Phase 05 demo chrome must not introduce additional font sizes.

## Demo Surface

### Terminal Output

The Playwright demo command prints the local viewer URL and deterministic fixture URL. CLI output is part of the UX contract because it is the first user-visible step.

Required lines:

```text
PhantomStream Playwright demo running on 127.0.0.1
Viewer: {viewerUrl}
Driven page: {drivenPageUrl}
Room: {roomKeyPrefix}...
Control: default-deny
```

Rules:
- Print the actual bound port after next-free-port fallback.
- Use `127.0.0.1` in URLs, not a LAN host.
- Do not auto-open browsers unless a CLI flag asks for it.
- Do not print a marketing paragraph or setup walkthrough.

### Page Shell

The first screen is the usable remote-control demo, not a landing page.

Required layout:
- Body: `background: #0f1117; color: #e0e0e0; padding: 24px; max-width: 1200px; margin: 0 auto`.
- Header: title left, compact lifecycle badge right, wraps cleanly under narrow widths.
- Subtitle: 15px body text, `#888888`, one short sentence.
- Main content: a compact status/authorization area above one large mirror stage. Do not use nested cards.
- Footer/log: mono, 13px, `#888888`, capped to 8 visible lines.

### Required Regions

| Region | Required Content | Rules |
|--------|------------------|-------|
| Header badge | `Connecting`, `Live`, `Stale`, or `Disconnected` | Mirrors viewer lifecycle exactly. Uses `role="status" aria-live="polite"`. |
| Status strip | lifecycle, room prefix, relay state, adapter state, last frame, navigation count | No mirrored content. Values use mono stack. |
| Authorization strip | authorization mode, control state, primary CTA, stop control button when active | Host-owned UI proving SEC-04. |
| Health row | frames, snapshots, mutations, misses, apply failures, drops, errors, control requests, denied, dispatched | Counters only; no payload data. |
| Mirror stage | sandboxed live mirror plus host-owned transparent control overlay | Primary visual anchor; stable dimensions. |
| Action log | last 8 authorization/control events | Must redact typed content and mirrored content. |

### Mirrored Fixture Content

The driven page is a local deterministic fixture, not a public website. It should contain visible targets that prove both automation-to-viewer and viewer-to-driver paths:
- A click target labeled `Click target` that increments a visible count.
- A text input labeled `Remote text` where remote typing is visible in the mirror.
- A scrollable/tall region with numbered rows and a visible scroll marker.
- A navigation/reload target labeled `Navigate fixture` to prove fresh snapshot after navigation.

The fixture may use the same visual tokens as the demo, but it is captured page content, not framework chrome. It must not contain secrets or user-private data.

## Mirror Stage Contract

The mirror stage is the primary visual anchor:

```css
.stage {
  position: relative;
  aspect-ratio: 16 / 10;
  min-height: 280px;
  height: min(72vh, 720px);
  background: #0f1117;
  border: 1px solid #333333;
  border-radius: 12px;
  overflow: hidden;
}
```

State rules:
- Before first snapshot, show the centered empty state behind the viewer root.
- On `live`, hide the empty state and show the mirrored page.
- On `stale` or `disconnected`, keep the last rendered frame visible if one exists; do not add a blocking full-stage overlay.
- On `control active`, add a `2px solid #f59e0b` stage border without resizing the stage. Use `outline` or inner overlay border, not layout-changing border width.
- On `denied`, do not alter the mirrored frame; show denial in the authorization strip and action log.
- On small screens, stage width remains `100%`; height floor may reduce to `220px`; no horizontal scrolling.

## Lifecycle & Authorization State Contract

Viewer lifecycle names are exact: `connecting`, `live`, `stale`, `disconnected`.

| State | Visual | Trigger Intent | Demo Copy |
|-------|--------|----------------|-----------|
| `connecting` | Attention dot `#eab308`; no pulse | Viewer or adapter waiting for first accepted frame | `Connecting` |
| `live` | Live dot `#22c55e`; pulse allowed when reduced motion permits | First accepted snapshot/frame has arrived and transport is healthy | `Live` |
| `stale` | Attention dot `#eab308`; no pulse | Last frame retained but freshness is suspect or navigation resnapshot is pending | `Stale` |
| `disconnected` | Destructive dot `#ef4444`; no pulse | Transport closed beyond the reconnect/stale window | `Disconnected` |

Remote-control state names are exact for the demo UI: `locked`, `requesting`, `active`, `denied`, `stopped`.

| State | Visual | Interaction | Required Event/Telemetry |
|-------|--------|-------------|--------------------------|
| `locked` | Muted dot `#6b7280`; no active border | `Request control` enabled only when lifecycle is `live`; overlay inert | state event with no content payload |
| `requesting` | Attention dot `#eab308`; CTA disabled as `Requesting control` | Waiting for host authorization hook | request counter increments |
| `active` | Live dot `#22c55e`; stage active border `#f59e0b`; badge text `Control active` | Overlay captures click, key, and wheel events; `Stop control` visible | approval counter increments; no mirrored content payload |
| `denied` | Destructive dot `#ef4444`; no active border | Overlay remains inert; no driver input dispatches | denial counter increments; denied state event emitted |
| `stopped` | Muted dot `#6b7280`; no active border | Overlay inert; primary CTA returns to `Request control` | stopped state event emitted |

Authorization control:
- Use a compact segmented control labeled `Authorization hook` with options `Approve` and `Deny`.
- Default mode is `Deny` so the default-deny path is immediately testable.
- `Request control` invokes the hook; it never bypasses the hook.
- When the mode is `Deny`, `Request control` must visibly enter `denied` and dispatch zero driver actions.
- When the mode is `Approve`, `Request control` enters `active`.

## Remote-Control Interaction Contract

### Control Overlay

- The host-owned transparent overlay sits above the sandboxed viewer iframe only while control is `active`.
- The overlay uses the current viewer scale, offset, viewport width, and viewport height to reverse-map host coordinates into driven-page viewport coordinates.
- Overlay event capture is limited to click/pointer down-up, printable text/key events, and wheel/scroll.
- The overlay must not dispatch synthetic DOM events into the captured page. Actions are replayed only through Playwright/CDP driver-native input.
- Keyboard capture requires the stage to be focused. The stage uses `tabindex="0"` and a visible focus ring.
- Pressing `Escape` stops control and returns to `stopped`; it is not a destructive action and needs no confirmation.

### Action Feedback

| Action | Immediate Feedback | Last Action Copy | Privacy Rule |
|--------|--------------------|------------------|--------------|
| Click | 2px accent ring at mapped host coordinate for 450ms | `Click sent: x {x}, y {y}` | Coordinates only; no element text/name |
| Type | Feedback chip at top-left of stage for 900ms | `Type sent: {n} chars` or `Key sent: {key}` | Printable text content is never displayed |
| Scroll | Brief accent side tick for 450ms in scroll direction | `Scroll sent: dx {dx}, dy {dy}` | Deltas only; no page content |
| Blocked | Destructive text in authorization strip | `Action blocked: control locked` | No driver input dispatched |

Feedback timing:
- Click ring and scroll tick may animate opacity for 450ms when `prefers-reduced-motion: no-preference`.
- Under reduced motion, show the same feedback statically for 450ms without movement.
- Feedback elements are absolutely positioned inside the stage and must not resize or shift the stage.

## Health & Logging Contract

Health row counters:
- `frames`
- `snapshots`
- `mutations`
- `misses`
- `apply failures`
- `drops`
- `errors`
- `control requests`
- `denied`
- `dispatched`
- `navs`

Rules:
- Counters update on viewer/adapter events or animation-frame batching; do not run visual timers while idle.
- Timestamps use local time and display `never` before first value.
- Error count is numeric; detailed errors go to browser console or test artifacts, not a content-bearing UI panel.
- Action log lines follow `{time} {event}` and cap at 8 entries.
- Logs may include event names, state names, counts, coordinates, and key names such as `Enter` or `Tab`.
- Logs must not include page HTML, attributes, captured text, input values, URLs beyond local demo URLs, or typed printable strings.

## Responsive & Accessibility Contract

Responsive:
- Header, status strip, authorization strip, health row, and controls use `display: flex; flex-wrap: wrap; gap: 8px`.
- Metric blocks use `min-width: 120px`; control buttons use `min-width: 132px`.
- The segmented authorization control uses stable option widths of at least `88px`; on narrow screens options may flex to equal widths.
- No nested cards, marketing hero, decorative gradient/orb backgrounds, or horizontal page scroll.
- Long log/status text wraps with `overflow-wrap: anywhere`; metric labels stay short enough to fit at 13px.

Accessibility:
- Lifecycle and control badges use `role="status" aria-live="polite"`.
- Status dots and feedback rings are `aria-hidden="true"`.
- `Request control` exposes `aria-disabled="true"` when lifecycle is not `live` or a request is pending.
- Authorization segmented options expose `aria-pressed` or native radio semantics; exactly one option is active.
- Stage focus ring is `2px solid #f59e0b`, offset `2px`.
- Color-coded states always include visible text labels.
