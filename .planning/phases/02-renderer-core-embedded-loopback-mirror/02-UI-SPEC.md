# Phase 2 — UI Design Contract

> Visual and interaction contract for the renderer core (`src/renderer/`) and the loopback demo (`examples/loopback-mirror.html`). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Binding context (02-CONTEXT.md):** Overlay visuals are REFERENCE PARITY with `reference/dashboard/dashboard.js` and FSB's `showcase/css/dashboard.css`. Layout modes are dropped. The only new design surface is the demo page. This document encodes the extracted reference values as the contract; every value is tagged with its provenance.

| Tag | Meaning |
|-----|---------|

## Component Contracts (Renderer Built-ins)

### Viewer root (`createViewer({ container, transport })`)

| Property | Contract | Provenance |
|----------|----------|------------|
| Root element | renderer-created `div` filling the container: `position: relative; width: 100%; height: 100%; overflow: hidden` | `[NEW — layout modes dropped per CONTEXT]` |
| Root background | transparent (letterbox bars show host background; host styles its own container) | `[NEW]` |
| Iframe sandbox | `sandbox="allow-same-origin"` ONLY — asserted at creation, fail loudly | `[PARITY dashboard.html + roadmap criterion 3]` |
| Iframe title | `"PhantomStream live mirror"` (reference: `"Live page preview"` — naming divergence, record in ledger) | divergence |
| Iframe position | `position: absolute; left: offsetX; top: offsetY; width: pageW px; height: pageH px; transform: scale(s); transform-origin: top left` | `[PARITY dashboard.js:2864-2868]` |
| z-order inside root | iframe `z-index: 1`; overlay layer `z-index: 2` | `[NEW — layer structure is discretion]` |

### Scale-to-fit (viewport-adaptive scaling)

Exact parity formula, simplified to single fill-container mode `[PARITY dashboard.js:2831-2869]`:

```
pageW = max(1, payload.viewportWidth  || 1920)
pageH = max(1, payload.viewportHeight || 1080)
s     = min(containerW / pageW, containerH / pageH)   // clamp: if !isFinite(s) || s <= 0 → s = 1
offsetX = max(0, (containerW - pageW * s) / 2)
offsetY = max(0, (containerH - pageH * s) / 2)
```

Recompute on `ResizeObserver` of the container `[PARITY dashboard.js:3200-3207, CONTEXT decision]`. Mirror is centered with symmetric letterboxing; aspect ratio always preserved.

### Mirror document (srcdoc) reset CSS

Injected into the snapshot HTML head — exact parity string `[PARITY dashboard.js:2795-2800]`:

```css
body { margin: 0; overflow: hidden; }
*::selection { background: transparent; }
::-webkit-scrollbar { display: none; }
```

Plus `<meta charset="UTF-8">` and `<meta name="viewport" content="width={viewportWidth|1920}">`. Full wrapper: `<!DOCTYPE html><html {htmlAttrs}><head>…stylesheetLinks…inlineStyleTags…resetStyle</head><body {bodyAttrs}>{payload.html}</body></html>`.

### Overlay layer (host document, never inside the iframe)

| Property | Contract |
|----------|----------|
| Element | `div` sibling above iframe inside viewer root: `position: absolute; inset: 0; overflow: hidden; pointer-events: none; z-index: 2` `[NEW]` |
| Sub-layering | glow `z-index: 10` → progress `z-index: 20` → dialog `z-index: 30` (preserves reference ordering: glow < dialog, dialog topmost `[PARITY: dialog z-15 was highest preview layer]`) |
| Coordinate mapping | `top = offsetY + y*s`, `left = offsetX + x*s`, `width = w*s`, `height = h*s` `[PARITY dashboard.js:3381-3384]` |
| Unknown overlay kinds | log via injected logger + ignore — never throw `[CONTEXT decision]` |
| Reset on new snapshot | glow, progress, dialog all hidden `[PARITY dashboard.js:2762-2764]` |
| Extension seam | `registerOverlay(kind, renderFn(payload, anchorRect, layer))`; built-ins registered through the same registry `[CONTEXT decision]` |
| Accessibility | overlay layer `aria-hidden="true"` except dialog card, which gets `role="status" aria-live="polite"` (additive divergence, record in ledger) |

### Action-glow rect (built-in `glow`)

Verified parity facts `[PARITY 122.1-VERIFICATION.md:98-110]`: orange-bordered rectangle, box-shadow glow, repositions with 100ms ease CSS transitions, `display: none` when inactive or `glow.state !== 'active'`.

Full contract (exact pixel values `[RECON]` — FSB css lines 1644-1653 not vendored):

```css
.ps-overlay-glow {
  position: absolute;
  border: 2px solid #f59e0b;
  border-radius: 4px;
  box-shadow: 0 0 12px rgba(245, 158, 11, 0.6);
  background: transparent;
  pointer-events: none;
  transition: top 100ms ease, left 100ms ease, width 100ms ease, height 100ms ease;
  display: none; /* shown only while glow.state === 'active' */
}
```

### Progress pill (built-in `progress`)

Text format — exact parity `[PARITY dashboard.js:3390-3402]`:
- determinate: `Math.round(percent) + '%' + ' - ' + phase` → e.g. `62% - Navigating`
- indeterminate: `(label || phase || 'Working') + ' - ' + (phase || 'Working')`
- hidden when payload has no `progress` field

Visual contract `[RECON — reference CSS not vendored; styled in the reference scrim/pill family]`:

```css
.ps-overlay-progress {
  position: absolute;
  bottom: 8px;
  left: 8px;
  background: rgba(0, 0, 0, 0.75);
  backdrop-filter: blur(4px);
  -webkit-backdrop-filter: blur(4px);
  color: #e0e0e0;
  font: 600 13px/1.2 system-ui, sans-serif;
  padding: 4px 12px;
  border-radius: 6px;
  max-width: calc(100% - 16px);
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
  pointer-events: none;
  display: none;
}
```

### Dialog card (built-in `dialog` — alert/confirm/prompt mirroring)

Exact parity — this CSS shipped verbatim in FSB (plan executed exactly as written, `[PARITY 124-02-PLAN.md:313-360 + 124-02-SUMMARY.md]`). Class names renamed `dash-preview-*` → `ps-overlay-*` only:

```css
.ps-overlay-dialog {           /* backdrop */
  position: absolute; top: 0; left: 0; right: 0; bottom: 0;
  background: rgba(0, 0, 0, 0.5);
  display: flex; align-items: center; justify-content: center;
  pointer-events: none;
  /* display: flex when open, none when closed */
}
.ps-overlay-dialog-card {
  background: #1e1e2e;
  border: 1px solid #333;
  border-radius: 12px;
  padding: 24px;
  max-width: 320px;
  width: 80%;
  text-align: center;
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
}
.ps-overlay-dialog-icon {
  font-size: 28px;      /* SVG: width/height 28px */
  color: #f59e0b;
  margin-bottom: 12px;
}
.ps-overlay-dialog-type {
  font-size: 13px; font-weight: 600;
  text-transform: uppercase; letter-spacing: 1px;
  color: #888;
  margin-bottom: 8px;
}
.ps-overlay-dialog-message {
  font-size: 14px; color: #e0e0e0; line-height: 1.5;
  word-break: break-word; max-height: 200px; overflow-y: auto;
}
```

Behavior `[PARITY dashboard.js:3405-3443]`:
- Type label = capitalized dialog type: `Alert` / `Confirm` / `Prompt` (default `Alert`)
- Icon by type: alert → warning-triangle SVG, confirm → question-circle SVG, prompt → keyboard SVG (inline SVG replaces Font Awesome — divergence, see Design System)
- `state === 'open'` → `display: flex`; `state === 'closed'` → `display: none`; hidden on new snapshot
- Message rendered via `textContent` (never innerHTML)

### Scroll mirroring

`[PARITY dashboard.js:3358-3372]`: store `scrollX/scrollY`, apply via `contentWindow.scrollTo({ left, top, behavior: 'smooth' })`; re-apply last scroll position after each mutation batch `[PARITY dashboard.js:3340-3342]`. Viewer-side scroll is read-only follow this phase (re-syncs to captured position) `[CONTEXT decision]`.

## Motion & Interaction Contract

| Interaction | Timing | Provenance |
|-------------|--------|------------|
| Glow reposition | 100ms ease on top/left/width/height | `[PARITY 122.1-VERIFICATION.md:104]` |
| Scroll follow | native `behavior: 'smooth'` | `[PARITY dashboard.js:3369]` |
| Demo LIVE-dot pulse | `@keyframes pulse { 0%,100% {opacity:1} 50% {opacity:.6} }`, 2000ms ease-in-out infinite | `[PARITY 122-02-PLAN.md:131-133 dashPreviewPulse]` |
| Demo button hover | background shift, 150ms ease | `[NEW, echoes reference 150ms toggle transition]` |
| Dialog open/close | instant show/hide (no animation) | `[PARITY — reference has none; do not add]` |
| Reduced motion | wrap pulse + glow transitions in `@media (prefers-reduced-motion: no-preference)` (demo + viewer) | `[NEW, additive]` |

Self-healing behavior visible to users: ≥3 stale-target misses or ≥2 apply failures trigger a `dash:request-snapshot` re-sync — the mirror repaints rather than drifting `[PARITY dashboard.js:3224-3335; CONTEXT decision]`. Miss accounting goes to the injected logger this phase (no UI surface — VIEW-02 is Phase 4).

## Demo Page Contract (`examples/loopback-mirror.html`) `[NEW]`

The project's "first light" demo. One HTML file, native ES module imports, zero dependencies, dark, clean.

**Layout:**
- Page: `background: #0f1117; color: #e0e0e0;` padding 24px; `max-width: 1200px; margin: 0 auto`
- Header: page title (28px/600) + subtitle (15px/400, `#888`) + LIVE badge; margin-bottom 24px
- Two-pane CSS grid: `grid-template-columns: 1fr 1fr; gap: 16px`; stacks to one column under 900px
- Each pane: `background: #1e1e2e; border: 1px solid #333; border-radius: 12px; padding: 16px`
- Left pane "SOURCE": a small self-mutating playground (list of rows + controls). Right pane "MIRROR": the viewer container, `aspect-ratio: 16 / 10; min-height: 200px` (echoes reference 16:10 stage and 200px floor `[PARITY dashboard.js:2837,2843]`)
- Pane headers: 13px/600 uppercase, letter-spacing 1px, `#888`, margin-bottom 8px

**LIVE badge:** inline flex, 13px/600 label "LIVE", preceded by an 8px round dot in `#22c55e` pulsing at 2000ms; turns `#6b7280` solid with label "PAUSED" when auto-mutate is off.

**Controls (left pane, gap 8px):**
- Primary button "Add row": `background: #f59e0b; color: #0f1117;` 600 weight, padding 8px 16px, radius 6px
- Secondary buttons "Remove row", "Edit text", "Show dialog", "Auto-mutate" (toggle): transparent background, `1px solid #333` border, `#e0e0e0` text, same padding/radius
- Hover: primary brightens (`filter: brightness(1.1)`); secondary background `rgba(255,255,255,0.06)`

**Behavior:** mirror starts automatically on load (criterion: `createViewer` auto-attaches). "Show dialog" calls native `alert()` to prove dialog mirroring. "Auto-mutate" toggles a ~1s interval of random row mutations.

**Footer line (13px mono, `#888`):** `Loopback transport — zero network, zero servers. Capture and viewer share this document.`
