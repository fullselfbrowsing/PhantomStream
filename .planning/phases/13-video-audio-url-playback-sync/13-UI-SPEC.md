# Phase 13 — UI Design Contract

> Visual and interaction contract for the **media-affordance overlay states** added to the renderer (`src/renderer/`) in Phase 13. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope is deliberately small.** Phase 13 is a media-playback-sync *protocol* phase. The only net-new UI is three renderer overlay states layered over a mirrored `<video>`/`<audio>`: **(A) a "blocked play" affordance** (shown when the in-iframe element's `play()` promise rejects under autoplay policy, clickable to start playback), **(B) an "unmute" affordance** (shown when the source is unmuted but the viewer started muted per the muted-autoplay default), and **(C) a poster-only rendering state** (`mediaMode === 'poster'`). These are overlays positioned in the renderer's PARENT realm over the sandboxed iframe (no scripts run in the iframe). They are a **sibling of the existing overlay system** (`src/renderer/overlays.js`: glow rect, progress pill, dialog card), not a new design language. This document does NOT invent a component library or redesign the viewer chrome.
>
> **Binding context:** All behavioral decisions are LOCKED in `13-CONTEXT.md` `<decisions>` Grey Area 4 (muted-autoplay default, `play()`-rejection affordance + `onMediaBlocked(nid)` host callback, `media-src` CSP add gated by the Phase 12 origin policy, `mediaMode` poster/reference split). This spec encodes the **visual + interaction contract for these affordance states only** and does not re-open them.

**Provenance legend (same convention as `02-UI-SPEC.md`):**

| Tag | Meaning |
|-----|---------|
| `[PARITY]` | Exact value reused from the shipped overlay system (`src/renderer/overlays.js` / `02-UI-SPEC.md`). Do not change — the media affordances must read as the same family. |
| `[CONTEXT]` | Locked by `13-CONTEXT.md` `<decisions>` Grey Area 4. Do not re-open. |
| `[NEW]` | New affordance detail (markup/copy/interaction) — was Claude's discretion per `13-CONTEXT.md` ("precise markup/styling of the click-to-play affordance, consistent with existing renderer overlay conventions"), now locked by this spec. |

## Affordance State Contract (Renderer)

The three states are overlays anchored to the mirrored media element's host-document rect via the existing nid→rect resolver (`resolveNidRect` / `OverlayContext`, `overlays.js`). They follow the overlay registry contract: registered through `register(kind, renderFn)`, rendered into `layer` (host document, `z-index: 2`, `pointer-events: none` at the layer), reset on new snapshot via `resetOverlays()`. Unlike the passive built-ins, the blocked-play and unmute affordances are **interactive**, so their own elements set `pointer-events: auto` (the layer stays transparent; only the clickable control opts in).

> **Security invariant (non-negotiable, `[PARITY]` with the overlay module):** all affordance text is written via `textContent`, never `innerHTML`. The only `innerHTML` assignments are the static inline-SVG glyph constants (the existing `ICON_SVG` precedent). No capture-influenced string is ever interpolated into markup. Affordances live in the host (parent) realm over the sandboxed iframe — they never enter the no-`allow-scripts` mirror document.

### State A — Blocked-play affordance (`play()` rejected) `[CONTEXT]`

Shown when the parent realm calls `.play()` on the in-iframe element and the returned promise **rejects** (autoplay blocked). `[CONTEXT: "play() rejection → observable affordance"]`

| Property | Contract | Provenance |
|----------|----------|------------|
| Trigger | `.play()` promise rejection on a mirrored media element (reconciler/playback driver catches it). | `[CONTEXT]` |
| Visual | A scrim (`rgba(0,0,0,0.5)`) clipped to the element's mapped rect, with a centered circular **play button**: amber play-triangle SVG (28px glyph), `2px solid #f59e0b` ring, `box-shadow: 0 0 12px rgba(245,158,11,0.6)`, transparent fill. | `[NEW]` + `[PARITY]` colors |
| Hit target | Button ≥ **44×44px** even over a small/audio element; centered in the rect. | `[NEW]` |
| Interaction | Button is `pointer-events: auto`; on activation, the renderer calls `.play()` on the in-iframe element again (now user-gesture-backed). On success the affordance hides. | `[CONTEXT]` |
| Keyboard / a11y | Button is a real focusable control (`role="button"`, `tabindex="0"`, `aria-label="Play mirrored media"`); Enter/Space activate it. The affordance container is **not** `aria-hidden` (it is actionable, unlike the decorative glow/progress). | `[NEW]` |
| Host callback | On entering this state, invoke the host hook `onMediaBlocked(nid)` (config-supplied function, same pattern as the Phase 12 `assetOriginPolicy` hook — **not** the `on('state'\|'health')` subscription, which throws on other event names). Errors from the hook are contained to the logger, never thrown. | `[CONTEXT]` + `[NEW: delivery mechanism]` |
| Reset | Hidden on new snapshot (`resetOverlays()` invokes the registered renderFn with a null payload, the universal hide/clear contract). | `[PARITY]` |
| Never | Never auto-clicks; never wedges the mirror (the rest of the mirror keeps updating behind the scrim). | `[CONTEXT: "mirror never wedges on a blocked play"]` |

### State B — Unmute affordance (source unmuted, viewer muted) `[CONTEXT]`

Shown when the source media is **unmuted** but the viewer started it **muted** to satisfy the muted-autoplay default. `[CONTEXT: "if the source is unmuted, surface an unmute affordance"]`

| Property | Contract | Provenance |
|----------|----------|------------|
| Trigger | Media is playing in the viewer with `muted === true` while the captured baseline/heartbeat reports the source as unmuted. | `[CONTEXT]` |
| Visual | A small **pill** anchored bottom-left of the element rect (mirrors the progress-pill anchor): `rgba(0,0,0,0.75)` background, `backdrop-filter: blur(4px)`, `border-radius: 6px`, padding `4px 12px`. Contents: an amber `#f59e0b` muted-speaker SVG (sized to the 13px text line) + 4px gap + label. | `[NEW]` + `[PARITY]` pill |
| Copy | Label: **`Unmute`** (13px/600, `#e0e0e0`). | `[NEW]` — see Copywriting |
| Interaction | Pill is `pointer-events: auto`; on activation the renderer sets the in-iframe element's `muted = false` (and restores the source's `volume` from the captured baseline). On success the pill hides. | `[CONTEXT]` |
| Keyboard / a11y | Focusable control (`role="button"`, `tabindex="0"`, `aria-label="Unmute mirrored media"`). | `[NEW]` |
| Reset | Hidden on new snapshot via `resetOverlays()`. | `[PARITY]` |

### State C — Poster-only rendering state (`mediaMode === 'poster'`) `[CONTEXT]`

When `mediaMode === 'poster'`, the viewer shows **the poster image only, no playback** (source not bound, autoplay disabled). `[CONTEXT: "poster = poster image only, no playback"]`

| Property | Contract | Provenance |
|----------|----------|------------|
| Rendering | The mirrored element shows its `poster` (already absolutified at capture; `poster` is in `URL_ATTRS`) fetched through the Phase 12 string-layer `gateAssetUrl` origin gate. No source/`<source>` URL is bound; no playback driver runs. | `[CONTEXT]` + Phase 12 gate |
| Affordance | **No interactive overlay** in poster mode — it is intentionally a still frame (no blocked-play, no unmute, since nothing plays). | `[CONTEXT]` |
| Optional caption | If, and only if, the element has **no** poster to show, a passive caption pill (same visual family as the unmute pill, but `pointer-events: none`, no accent, no icon) reads **`Media (poster only)`** centered in the rect so the region is not a confusing blank. Text via `textContent`. | `[NEW]` |
| Reset | Re-evaluated per snapshot like all other render state. | `[PARITY]` |

**`mediaMode === 'off'`** (for completeness — no UI surface): no media element fetch at all; the existing Phase 12 blocked-asset placeholder path (`data-ps-asset-unavailable`) already covers the visual. This phase adds nothing for `off`. `[CONTEXT]`

## Motion & Interaction Contract

| Interaction | Timing | Provenance |
|-------------|--------|------------|
| Affordance show/hide | instant (no animation) — matches the dialog/progress show-hide; do not add a transition | `[PARITY: "Dialog open/close: instant show/hide (no animation)"]` |
| Blocked-play button hover/focus | accent ring brightens (`filter: brightness(1.1)`), wrapped in `@media (prefers-reduced-motion: no-preference)` | `[NEW, additive — mirrors 02-UI-SPEC demo button + reduced-motion guard]` |
| Reposition on rescale | the affordance re-anchors to the element rect on `ResizeObserver`/resize, using the same coordinate math as overlays (`mapRectToHost`) | `[PARITY: overlay coordinate mapping]` |
| Reduced motion | any hover/focus transition guarded by `@media (prefers-reduced-motion: no-preference)`; no looping/auto motion anywhere | `[NEW, additive]` |

These affordances are **driven by renderer state**, not by a `STREAM.MEDIA` cadence — the wire channel carries playback state to the reconciler; the affordance appears/disappears as a function of local `play()` outcome / `muted` mismatch / `mediaMode`. (The `STREAM.MEDIA` op shape, throttle, and reconciler thresholds are protocol contracts, out of scope for this visual spec.)

## Copywriting Contract

Voice: declarative, lowercase-tech, no exclamation marks — same register as `02-UI-SPEC.md` ("zero network"). Labels are short verbs/noun-phrases; all rendered via `textContent`.

| Element | Copy |
|---------|------|
| Primary CTA (blocked-play) | **Icon-only** play button — `aria-label="Play mirrored media"` (no visible text label; the accent play glyph is the affordance). |
| Unmute affordance label | `Unmute` (`aria-label="Unmute mirrored media"`) |
| Poster-only caption (no-poster fallback only) | `Media (poster only)` |
| Empty / waiting state | none new this phase — pre-snapshot waiting state is owned by `02-UI-SPEC.md`; a media element with no source simply shows its poster or the Phase 12 placeholder. |
| Error state | none new this phase — a blocked `play()` is **not** an error (it is the expected autoplay path and is handled by State A); a blocked media **URL** reuses the Phase 12 `data-ps-asset-unavailable` placeholder. No new error UI. |
| Destructive confirmation | **none** — there are no destructive actions in these affordances (clicking play or unmute is non-destructive and self-reversing). |
