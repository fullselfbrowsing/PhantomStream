## Phase Goal

**As a** host embedding the PhantomStream viewer, **I want to** mirror a `<video>`/`<audio>` element's playback (play/pause/seek/rate) from the captured tab over a new throttled side channel with drift-corrected sync, **so that** the viewer plays the source media in lockstep without the bytes ever crossing the relay and without wedging on autoplay policy.

<objective>
The capture slice: read live media element state into a `media[]` snapshot baseline and emit nid-addressed `STREAM.MEDIA` messages on real media events plus a throttled `timeupdate` heartbeat — the source-side half of the sync pipeline. After this slice, a real captured `<video>` produces actual `STREAM.MEDIA` traffic and a real snapshot `media[]` baseline that the renderer slice (Plan 03) and the oracle slice (Plan 04) consume.

Purpose: Capture is the only tier that sees the real source `HTMLMediaElement`. This slice produces the wire data the rest of the phase reacts to — without it the renderer driver has nothing to drive and the oracle has no divergence to ledger.
Output: `startMediaTracker()`/`stopMediaTracker()`, the `media[]` baseline in `serializeDOM`, per-element listeners wired into the added-node loop, and the capture-media test suite.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| captured page DOM -> capture core | The `<video>`/`<audio>` elements and their property values are attacker-influenceable; capture reads them but must not let them mutate capture state or the page. |
| capture core -> wire (STREAM.MEDIA / media[]) | Emitted playback state crosses into the raw relay; it must be nid-addressed text within the 1 MiB cap, never media bytes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering | media[] baked into serialized HTML clone | mitigate | media state is read from the LIVE element and written only to snapshotPayload.media; a test asserts payload.html and the live element outerHTML are unchanged (Phase 7 no-mutation invariant + differential-oracle HTML byte-identity). |
| T-13-05 | Information Disclosure | media BYTES on the relay | mitigate | Capture emits only nid-addressed playback STATE (numbers/booleans/event name); no src bytes, no media blob ever enters safeSend. The low-bandwidth core value (bytes via the viewer's own network) is preserved by construction. |
| T-13-06 | Denial of Service | unthrottled timeupdate flooding the 1 MiB-capped relay | mitigate | timeupdate is throttled at MEDIA_SYNC_THROTTLE_MS and only while playing; discrete transitions are bounded (one per real DOM event); per-element lastMediaSend prevents a hot loop. |
| T-13-07 | Spoofing | late media frames from a prior page/session | mitigate | Every STREAM.MEDIA payload is stamped with the current streamSessionId + snapshotId so the renderer's isCurrentStream guard (Plan 03) rejects cross-generation frames. |
| T-13-SC | Tampering | npm/pip/cargo installs | mitigate | Zero packages installed (RESEARCH audit: none). No install task; no blocking checkpoint required. hls.js is Phase 14 and absent here. |
