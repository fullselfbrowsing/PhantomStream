<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wire CSS source -> mirror iframe head | Page-controlled CSS text and hrefs become style/link elements in the mirror document. |
| wire CSS source -> reconstructed shadow root | Page-controlled CSS text is inserted into a shadow root created by the renderer. |
| wire CSS source -> nested frame srcdoc | Page-controlled CSS text is inserted into an inert frame document. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-13 | E | `buildSnapshotHtml` | mitigate | Keep CSP meta and existing iframe sandbox contract; do not introduce `allow-scripts` or executable style handling. |
| T-09-14 | T | `setScopedStyleText` | mitigate | Use `scrubCssText` and `textContent`, never `innerHTML`, for CSSOM text insertion. |
| T-09-15 | I | scoped install warnings | mitigate | Warnings include only sourceId/scope kind/reason, not CSS text, selectors, hrefs, or page content. |
| T-09-16 | T | href style sources | mitigate | Reuse dangerous stylesheet URL filtering before inserting CSSOM href links. |
</threat_model>
