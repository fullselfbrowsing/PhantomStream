<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page stylesheet/CSSOM -> capture serializer | Page-controlled CSS sources are read and represented as wire payloads. |
| cross-origin sheet -> fallback chain | Browser origin policy blocks cssRules and must not be bypassed. |
| host hook -> capture core | Explicit `fetchStylesheet` returns host-provided CSS text that still requires sanitization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-09 | I | `collectCssomStyleSourcesForScope` | mitigate | Catch `cssRules` errors per sheet; use safe href relink, explicit hook, or computed fallback without logging or transporting blocked CSS text. |
| T-09-10 | T/E | `sanitizeCssTextForSource` | mitigate | Route all readable, fetched, constructable, inline, and fallback CSS through `sanitizeForWire('css', ...)`. |
| T-09-11 | D | computed fallback | mitigate | Use existing curated computed props only after fallback selection; tests forbid broad enumeration in normal CSSOM path. |
| T-09-12 | I | diagnostics | mitigate | Emit content-free reason identifiers only: `cssRules-blocked`, `href-relinked`, `adapter-fetch`, `computed-fallback`. |
</threat_model>
