<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mutation payload -> renderer diff applier | Style-source ops from capture request CSS changes in the mirror. |
| renderer identity map -> scoped style root | Style ops rely on nid-indexed host/frame resolution. |
| dynamic CSS text -> style element | Replacement CSS text is inserted after scrub into existing mirror scopes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-21 | T | `applyStyleSourceOp` | mitigate | Validate action/sourceId/scope and delegate only to scoped hooks; no selector fallback to page-owned identity attributes. |
| T-09-22 | R | `handleMutations` | mitigate | Keep existing `isCurrentStream(payload, active)` guard before style op application. |
| T-09-23 | I | stale style scope handling | mitigate | Resync reason `stale-style-scope` carries only sourceId and scope kind, not CSS or URLs. |
| T-09-24 | E | dynamic style insertion | mitigate | Use `setScopedStyleText`/`scrubCssText` and existing sandbox/CSP; no `allow-scripts`. |
</threat_model>
