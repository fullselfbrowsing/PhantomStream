<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page CSSOM -> capture payload | Untrusted page-authored CSS, CSSOM rule text, and constructable stylesheet text enter the wire payload. |
| cross-origin stylesheet -> fallback metadata | Browser-blocked stylesheet contents must not be read, fetched, logged, or transported without explicit host capability. |
| capture payload -> renderer style insertion | CSS text from the wire is inserted into the sandboxed mirror document, shadow roots, or frame documents. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | I | CSSOM fallback tests | mitigate | RED tests require per-sheet `cssRules` failures to avoid content leakage and use safe href relink, explicit `fetchStylesheet`, or computed fallback. |
| T-09-02 | T/E | CSS text sanitizer tests | mitigate | RED tests require capture `sanitizeForWire('css', ...)` and renderer `scrubCssText` for all CSSOM CSS text paths. |
| T-09-03 | I | diagnostics tests | mitigate | RED tests require content-free reason identifiers and forbid CSS text, selectors, hrefs, URLs, and page text in diagnostics. |
| T-09-04 | D | no-broad-enumeration tests | mitigate | RED tests require CSSOM normal path to avoid broad computed-style enumeration and allow curated fallback only after the fallback chain selects it. |
</threat_model>
