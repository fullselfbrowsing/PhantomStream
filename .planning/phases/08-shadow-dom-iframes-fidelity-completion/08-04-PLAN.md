<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shadow DOM -> capture serializer | Component-owned shadow DOM may contain hostile markup or private form content. |
| shadow payload -> renderer shadow root | Serialized shadow HTML is parsed and inserted into a real mirror shadow root. |
| shadow identity -> renderer Map | Shadow descendants become addressable through Phase 7 private identity state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-13 | E | `installShadowRoots` | mitigate | Parse into template, run `sanitizeFragment`, then import/index; never install unsanitized shadow HTML. |
| T-08-14 | T | shadow identity sidecars | mitigate | Use `ensureNodeId`/`nodeIds` and renderer private Map; no selector or XPath fallback. |
| T-08-15 | I | slotted content serialization | mitigate | Preserve slot elements only in shadow HTML and leave assigned light DOM children in normal payload HTML. |
| T-08-16 | D | shadow observer expansion | mitigate | Observe discovered open roots through the existing rAF batcher and contain attachShadow wrapper errors in logger. |
</threat_model>
