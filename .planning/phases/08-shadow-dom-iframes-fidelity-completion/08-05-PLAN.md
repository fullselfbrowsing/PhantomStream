<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent document -> frame document | Capture may inspect only same-origin iframe documents exposed by the browser. |
| frame payload -> renderer nested iframe | Serialized frame content is parsed into nested inert mirror frames. |
| cross-origin iframe -> metadata label | Only safe metadata crosses the boundary for inaccessible frames. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-17 | I | `classifyFrame` | mitigate | Use `contentDocument` as the only same-origin gate; catch failures and emit content-free cross-origin metadata. |
| T-08-18 | E | `installFrames` | mitigate | Render frame payloads through CSP srcdoc and sandbox without `allow-scripts`; sanitize before indexing. |
| T-08-19 | T | frame identity | mitigate | Use `frameNid` and frame `nodeIds` sidecars with the renderer Map; no global selector addressing. |
| T-08-20 | R | frame diagnostics | accept | Runtime frame-access logs are content-free reason strings; no PII or mirrored text is logged. |
</threat_model>
