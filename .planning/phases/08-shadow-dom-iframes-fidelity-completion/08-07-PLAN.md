<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| viewer control -> capture | Viewer requests a live subtree by nid and active stream identity. |
| capture live DOM -> subtree response | Capture serializes attacker-controlled DOM after the initial snapshot. |
| subtree response -> renderer insertion | Renderer parses and installs returned HTML into the existing mirror. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-25 | D | `requestSubtree` | mitigate | Latch requests by nid/requestId; clear on response; ignore duplicate calls before response. |
| T-08-26 | I | `serializeRequestedSubtree` | mitigate | Return content-free statuses for stale/gone/skipped/blocked/untracked nodes and preserve masking. |
| T-08-27 | E | `handleSubtreeResponse` | mitigate | Parse to template, run `sanitizeFragment`, import, then index; never trust response HTML directly. |
| T-08-28 | T | staleness checks | mitigate | Require `streamSessionId` and `snapshotId` on request/response; use `isCurrentStream` before mutating mirror. |
</threat_model>
