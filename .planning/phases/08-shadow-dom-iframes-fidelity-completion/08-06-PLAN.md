<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| form controls -> wire value diff | User-entered values can contain secrets and must be masked capture-side. |
| computed style -> add-op HTML | CSS values come from the page and are rendered by the viewer. |
| value op -> renderer DOM property | Renderer applies transport data to form controls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-21 | I | `buildValueDiff` | mitigate | Route values through `sanitizeForWire('input')`, preserving password/maskInputs/maskInputFn behavior. |
| T-08-22 | R | value telemetry | mitigate | Do not add typed values to logger, health events, or diagnostics; tests scan whole wire. |
| T-08-23 | E | add-op styles | mitigate | Reuse `sanitizeForWire('subtree')` and CSS scrubbers after style insertion. |
| T-08-24 | D | `collectSubtreeComputedStyles` | mitigate | Read curated style props in a batch and reject broad property enumeration. |
</threat_model>
