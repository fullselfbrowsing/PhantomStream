<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> capture value op | Typed values may contain private data and must be masked before transport. |
| computed style -> wire HTML | Browser-computed CSS values are attacker-influenced and rendered in the mirror. |
| mutation stream -> renderer properties | Renderer receives value ops from transport and mutates form-control properties. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | I | capture value tests | mitigate | Tests must prove password, `maskInputs`, and `maskInputFn` apply to event-driven value diffs and raw typed text is absent from the whole wire. |
| T-08-06 | E | added style tests | mitigate | Tests must require existing CSS scrubbers to neutralize dangerous computed style values before renderer insertion. |
| T-08-07 | D | style performance tests | mitigate | Tests must detect broad all-property enumeration and require bounded `getComputedStyle` calls. |
| T-08-08 | T | renderer value op tests | mitigate | Tests must apply values through DOM properties and existing identity hooks, not attributes or selector lookup. |
</threat_model>
