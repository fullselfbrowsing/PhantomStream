## Decisions Made
- **RED via dynamic import() inside test bodies.** A top-level static ESM import of a missing named export is a link-time SyntaxError ("does not provide an export named ..."), and a top-level import of a missing module is a module-load error -- both would fail the whole file as a load/syntax error, which the plan explicitly says to avoid. Importing the target module inside each `test()` and asserting the (currently `undefined`) export is a function (or catching `ERR_MODULE_NOT_FOUND` and re-asserting) makes each case a clean per-test FAILURE while the file parses fine. This is the only pattern that satisfies BOTH the plan's parse-check `<verify>` AND its "failing tests, not a syntax error" acceptance criterion.
- **Oversized data: URI sizing.** Body padded to 280000 base64 chars (full URI utf8 = 280022 B) so it is comfortably > the 262144 B (`ASSET_DATA_URI_MAX_BYTES`) cap Plan 02 introduces; generated by a one-shot Node writer for a deterministic, reproducible fixture.
- **CSP is confirm-only.** No edit to `CSP_META` (`src/renderer/snapshot.js` verified byte-unchanged); the assertion is a string check because jsdom does not enforce meta-CSP -- real CSP enforcement is the Playwright UAT's job (Phase 12 manual-only verification, may be deferred).
