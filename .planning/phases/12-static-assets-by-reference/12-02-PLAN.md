<objective>
Close the two genuine capture-side gaps (ASST-04 non-shareable degrade; ASST-03 currentSrc
variant pin) so that, after this plan, a developer can capture a page with a responsive image
and a blob:/oversized-data image and SEE on the wire: the responsive image carries
`data-ps-currentsrc` (clone only), the blob:/oversized images become dimensioned placeholders,
small inline images stay byte-identical, and the differential oracle is green with the D26
divergence firing. Vertical slice: capture emits the hardened payload end-to-end and the oracle
proves it, in one `npm test`.

through, so non-shareable refs must be detected and degraded; `currentSrc` pinning is net-new
(0 hits in repo). Both are modeled on shipped precedents (`createBlockPlaceholder`,
`replaceWithBlockPlaceholder`, the nid clone-only discipline) so the truncation/oracle machinery
stays consistent.
Output: the byte-cap constant, three pure-ish capture helpers + their hooks at the four
serialization sites, a green capture-degrade suite, and the static-assets oracle scenario + D26
ledger entry.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| live DOM → wire clone | Capture must enrich/degrade on the detached clone only — never mutate the observed page |
| asset attribute value → wire | blob:/oversized-data must never be emitted as a fetchable reference |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03 | Denial of Service | capture `<img>` blob:/data: handling | mitigate | `blob:`/origin-local detected at capture → dimensioned placeholder (never a dead ref on the wire that breaks the mirror); `data:` over `ASSET_DATA_URI_MAX_BYTES` → placeholder so one inline image cannot blow the 1 MiB per-message cap. Task 1 hooks all four serialization sites. |
| T-12-04 | Tampering / Information Disclosure | live-page mutation via framework attrs | mitigate | `data-ps-currentsrc` and the asset-unavailable placeholder are written on the wire CLONE only (Pitfall 4 — the added-node path's wireClone is the trap); a capture test asserts the live DOM has zero `data-ps-*` attributes, preserving the Phase 7 no-mutation invariant so the page's own selectors/observers are unaffected. |
| T-12-05 | Tampering / Elevation | injection vs fetchability conflation | mitigate | The degrade is a *fetchability* control distinct from `hasDangerousScheme` (injection). `data-ps-currentsrc` is only set when `!hasDangerousScheme(resolvedCurrent)`, so an enriched currentSrc can never reintroduce a `javascript:`/`data:text/html` value the injection sanitizer would block. |
| T-12-SC | Tampering | npm/pip/cargo installs | mitigate | Phase 12 installs NO packages; this plan adds only in-repo plain-JS ESM. No install task → supply-chain checkpoint N/A by construction. |
