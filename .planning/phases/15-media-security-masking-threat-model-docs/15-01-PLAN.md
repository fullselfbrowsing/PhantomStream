## Phase Goal

**As a** host embedding the PhantomStream viewer to mirror attacker-influenced pages, **I want to** redact, strip, or block the asset/media URLs that leave my page (signed-CDN tokens, sensitive media) and minimize what the viewer-side fetch leaks, **so that** I can mirror real media by reference without exposing CDN credentials, private media timelines, or the mirrored page URL to third-party origins — and never weaken the sandbox.

> Derived from 15-CONTEXT `<decisions>` (the host masking vocabulary) + REQUIREMENTS MSEC-03/MSEC-04; the ROADMAP `**Goal**` line is outcome-shaped, so this user story is grounded in those locked artifacts, not invented. The ROADMAP goal verbatim: "Close the milestone by completing the security contract that was threaded through Phases 12-14 — full asset/media URL masking + referrerpolicy=no-referrer, a threat-review of the parent-realm object-URL blast radius, media-specific security tests, and the SECURITY/ARCHITECTURE documentation updates."
>
> **This plan's slice:** the capture masking spine — the host gains `maskMediaSelector`/`maskAssetUrls`/`maskAssetUrlFn` and a masked `<video>` emits nothing on the wire (the only genuinely new behavior; everything else completes/tests/documents it).

<objective>
Build the capture-side asset/media URL **masking vocabulary** (MSEC-03): three new host config options
(`maskMediaSelector`, `maskAssetUrls`, `maskAssetUrlFn`) routed through a single new `'asset-url'`/`'media-url'`
dispatch in the existing `sanitizeForWire` chokepoint, backed by one pure, unit-testable helper
(`maskAssetUrlForWire`) and a documented `TOKEN_PARAM_DENYLIST`. Wire `maskMediaSelector` into the already-shipped
media-tracker skip predicates so a masked `<video>`/`<audio>` emits **no** `STREAM.MEDIA` and degrades to the
existing dimension-only placeholder. This is the thickest vertical slice of the phase — the only genuinely new
behavior; everything downstream (referrer meta, threat-model docs, security tests) completes or documents it.

Purpose: A host that mirrors signed-CDN URLs (which carry tokens/PII) or sensitive media can now redact, strip, or
block those URLs **before transport**, fail-closed, without breaking the differential oracle (off-by-default keeps
the wire byte-identical).

Output: `src/capture/index.js` extended (config + helper + dispatch + predicate wiring); a new
`tests/security-asset-url-mask.test.js`; an extended `tests/capture-media.test.js` (masked-media-emits-no-state).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mirrored page (attacker-influenced) -> capture clone | Hostile attribute values (`src="javascript:..."`, signed-CDN tokens, sensitive media URLs) cross here; `sanitizeForWire` is the chokepoint. |
| capture -> wire | Only already-masked/stripped/scheme-scrubbed URL strings + small playback state leave; the relay carries no bytes. |
| host config -> capture (`maskAssetUrlFn`) | Host-supplied redactor runs in the capture realm; must be fail-closed-contained so a buggy/throwing host fn cannot wedge capture or leak a raw URL. |
