## Phase Goal

**As a** host embedding the PhantomStream viewer to mirror attacker-influenced pages, **I want to** redact, strip, or block the asset/media URLs that leave my page (signed-CDN tokens, sensitive media) and minimize what the viewer-side fetch leaks, **so that** I can mirror real media by reference without exposing CDN credentials, private media timelines, or the mirrored page URL to third-party origins — and never weaken the sandbox.

> Derived from 15-CONTEXT `<decisions>` (the host masking vocabulary) + REQUIREMENTS MSEC-03/MSEC-04; the ROADMAP `**Goal**` line is outcome-shaped, so this user story is grounded in those locked artifacts, not invented. The ROADMAP goal verbatim: "Close the milestone by completing the security contract that was threaded through Phases 12-14 — full asset/media URL masking + referrerpolicy=no-referrer, a threat-review of the parent-realm object-URL blast radius, media-specific security tests, and the SECURITY/ARCHITECTURE documentation updates."
>
> **This plan's slice:** the viewer-fetch leakage control — one document-level `<meta name="referrer" content="no-referrer">` so the host's mirrored page URL never leaks in a Referer header, with the omit-credentials posture asserted.

<objective>
Complete the MSEC-04 viewer-fetch leakage control: inject a single document-level
`<meta name="referrer" content="no-referrer">` into the renderer srcdoc, immediately after the CSP meta, so **no**
referrer (which can carry the mirrored page URL and its tokens) leaks to third-party CDNs on any viewer-side
subresource fetch — `<img>`, `<video>`/`<source>`, `<video poster>`, CSS `background-image`/`url()`, fonts.
Confirm-and-assert the already-shipped no-credentials posture (sandboxed `allow-same-origin` srcdoc + no `crossorigin`
attribute) by pinning the absence of any `crossorigin` attribute. One line of production code; the rest is the string
contract test.

Purpose: A mirrored page URL that itself carries a signed token must never appear in a `Referer` header sent to the
asset's origin. The document `<meta>` is the single, broadest control (it also covers CSS-initiated fetches that no
per-element `referrerpolicy` could reach).

Output: `src/renderer/snapshot.js` (the referrer meta after `CSP_META` at both return sites);
