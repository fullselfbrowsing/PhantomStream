    - `runScenario(side, scenario)`: starts the side, awaits `scenario.run(side, settle)` to FULL completion, stops, returns `side.sent`. Callers run side A to completion before constructing/running side B — never interleave (Pitfall 10). Wrap in try/finally calling `side.close()`.

    **tests/differential/normalize.js** — pure transforms, JSDoc'd, style of src/protocol/messages.js:
    - Import `STREAM` from `../../src/protocol/messages.js` (never restate `ext:` literals).
    - `normalizeReference(msgs)`: maps each `{ action, ...rest }` to canonical `{ type, payload }` using the fixed table: domStreamSnapshot→STREAM.SNAPSHOT (payload = msg.snapshot), domStreamMutations→STREAM.MUTATIONS (payload = { mutations, streamSessionId, snapshotId, staleFlushCount }), domStreamScroll→STREAM.SCROLL, domStreamOverlay→STREAM.OVERLAY, domStreamDialog→STREAM.DIALOG (payload = { dialog }), domStreamReady→STREAM.READY (payload = {}).
    - `canonicalizeIdentity(normalizedMsgs)`: replaces `streamSessionId` string values with ordinal placeholders by first occurrence (`SESSION_1`, `SESSION_2`, ...) and `snapshotId` numeric values with `SNAPSHOT_1`, `SNAPSHOT_2`, ... — ordinal (NOT constant) placeholders preserve the same-vs-fresh identity structure, which D-02 requires and which the pause/resume divergence (D1, Plan 01-04) depends on. Apply recursively wherever the fields appear (top-level payload and nested, e.g. dialog payloads carry identity via attachStreamMetadata). Do NOT canonicalize nids — both sides assign nids in identical TreeWalker document order; raw nid equality is signal (RESEARCH anti-pattern "normalizing too aggressively").
    - `compareStreams(refNormalized, otherNormalized, fixture, scenario, ledger)`: per-index `assert.deepStrictEqual` over `Math.max` of both lengths (missing index = undefined side); on mismatch, consult `ledgerCovers(ledger, refMsg, otherMsg, scenario)` — if uncovered, throw `new Error('UNDECLARED DIVERGENCE <fixture>/<scenario> at message <i>:\n' + originalError.message)`. Return the Set of ledger entry ids that matched (consumed later by stale-entry detection in Plan 01-04).

    **tests/differential/divergence-ledger.js** — machine-readable per D-03, style of grouped protocol consts: `export const DIVERGENCES = []` (empty at this point — reference-vs-reference has zero divergences by construction) plus JSDoc `@typedef DivergenceEntry` documenting the schema: `{ id, kind: 'mismatch' | 'documented-mapping', description, rationale, affectedMessages, affectedScenarios, appliesTo(refMsg, extMsg, scenarioName) }`. `kind: 'documented-mapping'` entries document divergences absorbed structurally by normalize.js and are exempt from stale-entry detection; `kind: 'mismatch'` entries must match at least one real divergence. Export `ledgerCovers(ledger, refMsg, extMsg, scenarioName)` returning the matching entry id or null (only consults `kind: 'mismatch'` entries).

Every test wraps sides in try/finally close() (Pitfall 3).
