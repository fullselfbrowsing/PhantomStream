## Decisions Made

- **Fixture-agnostic structural-ops:** the matrix runs structural-ops against both basic.html and heavy-realistic.html; generic selectors (`querySelector('ul'/'img'/'p')`, `querySelectorAll('li')[1]`) avoid duplicating the scenario per fixture and keep both fixtures free of artificial shared IDs
- **Guards read MATRIX:** truncation/dialog guard tests fetch their pair config from the MATRIX table rather than restating it -- structurally prevents cross-test config drift (Pitfall 6) and satisfies the exactly-one-`dangerously` acceptance grep
- **Memoized capturePair:** the 1.27M-char truncation fixture serializes exactly twice per suite run (matrix pair shares captures with its guard), keeping the suite at ~6.5s
- **Temp generator for heavy-realistic deleted pre-commit:** the plan freezes the fixture file as the artifact; committed generator provenance is required only for truncation-overflow

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] mutation-burst settles twice (plan specified one settle) to absorb the nid-echo flush**
- **Found during:** Task 2 (scenario authoring, pre-implementation analysis)
- **Issue:** The burst's 20 appended elements get data-fsb-nid stamped during the rAF flush; the observer reports those stamps as fresh attribute mutations flushed on the NEXT frame (~16 ms), while settle()'s tail is 20 ms -- a ~4 ms jitter margin where the echo batch could merge differently per side, exactly the flake class Plan 01-01 documented and fixed in basic-mutations
- **Fix:** mutation-burst (and structural-ops after its add group) settle twice, deterministically isolating the echo flush as its own message before the scenario proceeds
- **Files modified:** tests/differential/scenarios/mutation-burst.js, tests/differential/scenarios/structural-ops.js
- **Verification:** 3 consecutive full-suite runs all 21/21 with stable message shapes
- **Committed in:** 8945d08 (Task 2 GREEN commit)
