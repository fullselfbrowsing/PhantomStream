<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Content script -> service worker | Page-observed DOM data crosses into extension context. |
| Service worker -> relay transport | Stream frames leave extension process through WebSocket transport. |
| Service worker restart -> storage.session | Recovery state crosses an ephemeral runtime boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03 | Information Disclosure | `chrome.storage.session` | mitigate | Persist only content-free stream intent; tests reject payload-like keys. |
| T-06-04 | Denial of Service | watchdog alarm | mitigate | Alarm handler is idempotent and requests a fresh snapshot rather than replaying buffered diffs. |
| T-06-05 | Spoofing | runtime messages | mitigate | Message handler accepts only structured `{ type, payload }` bridge messages and validates known control/stream paths. |
| T-06-06 | Tampering | SW module globals | mitigate | Tests recreate adapter from storage and prove old module variables are unnecessary. |
</threat_model>
