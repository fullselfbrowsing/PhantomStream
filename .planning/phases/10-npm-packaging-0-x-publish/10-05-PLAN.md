    - `docs/QUICKSTARTS.md`
  </read_first>
  <action>
    Create `.github/workflows/publish.yml` with:
    - `name: Publish`
    - trigger on `release: types: [published]` and `workflow_dispatch`;
    - `permissions: { contents: read, id-token: write }`;
    - one job on `ubuntu-latest`;
    - `actions/checkout@v6`;
    - `actions/setup-node@v6` with `node-version: 24`, `registry-url: 'https://registry.npmjs.org'`, and `package-manager-cache: false`;
    - `npm ci`;
    - `npm run package:check`;
    - `npm publish --access public` as the final publish command.

    Do not add `NPM_TOKEN` or token secrets.

    Add `docs/RELEASE.md` documenting:
    - local preflight commands (`npm ci`, `npm run package:check`, `npm publish --dry-run --access public`);
    - npm trusted publisher setup fields for GitHub Actions;
    - that real publish requires package write permission, 2FA/trusted publishing, and explicit approval;
    - how to use staged publishing if the user wants review before public release.

    Add `publishConfig.access = "public"` to `package.json`.
  </action>
  <verify>
    <automated>rg -n "id-token: write|registry-url: 'https://registry.npmjs.org'|npm run package:check|npm publish --access public|NPM_TOKEN" .github/workflows/publish.yml docs/RELEASE.md package.json && ! rg -n "NPM_TOKEN" .github/workflows/publish.yml && npm run package:check</automated>
  </verify>
  <acceptance_criteria>
    - `test -f .github/workflows/publish.yml && test -f docs/RELEASE.md` passes.
    - `rg -n "id-token: write|npm publish --access public|npm run package:check" .github/workflows/publish.yml` returns matches.
    - `! rg -n "NPM_TOKEN" .github/workflows/publish.yml` exits 0.
    - `node -e "const p=require('./package.json'); if(!p.publishConfig || p.publishConfig.access !== 'public') process.exit(1)"`
    - `npm run package:check` exits 0.
  </acceptance_criteria>
  <done>Release workflow and docs are provenance-ready and token-free.</done>
</task>
